# 比較

この表は、`@lazarv/react-server`を他の人気のあるReactフレームワークおよびルーティングソリューションと比較しています。フルスタック機能、アーキテクチャ、ルーティング、型安全性、検索パラメータ、スクロール復元、開発者体験を網羅しています。

> この比較は`@lazarv/react-server`チームによって管理されています。不正確な点を見つけた場合は、[issueを開いてください](https://github.com/lazarv/react-server/issues) — 公平で正確な内容を目指しています。

**凡例：**

- ✅ — ファーストクラスの組み込みサポート
- 🟡 — 部分的または限定的なサポート
- 🔶 — コミュニティプラグイン、回避策、または手動セットアップで可能
- 🛑 — 非サポート

<Link name="notes">
## 注記
</Link>

`@lazarv/react-server`は完全なReact Server Componentsランタイムであり、単なるルーターではありません。ルーティングシステムはRSCストリーミング、サーバー関数、Viteビルドパイプラインと深く統合されています。これにより、型付きルート、クライアント専用ルート、サーバーサイドバリデーションなどの機能が、グルーコードなしでエンドツーエンドで動作します。

### 主なアーキテクチャの違い

- **RSCネイティブ**: RSCサポートを後付けするルーターとは異なり、`@lazarv/react-server`はReact Server Componentsのために一から構築されています。すべてのルートでサーバーコンポーネントとクライアントコンポーネントを自由に混在させることができます。
- **クライアント専用ルート**: ファイルシステムルーターで`"use client"`を持つページは自動的にクライアント専用ルートになります — ナビゲーションはサーバーを完全にスキップします。設定は不要で、Reactの`<Activity>`コンポーネントによりコンポーネントの状態はナビゲーション間で保持されます。
- **スキーマ非依存のバリデーション**: ルートパラメータと検索パラメータは、任意のスキーマライブラリ（Zod、ArkType、Valibot）または軽量パース関数でバリデーションできます — ランタイムがバリデーション戦略を自動的に検出します。
- **RSCがルートローダー**: `@lazarv/react-server`では、React Server Components*がルートローダーそのもの*です。各サーバーコンポーネントは`async/await`で独自のデータを取得します — 別のローダーAPI、ウォーターフォールチェーン、`useLoaderData`フックは不要です。RSCのストリーミングモデルと自動Suspenseバウンダリにより、追加の抽象化レイヤーなしで同じ遅延ロード体験を提供します。`"use cache"`によるstale-while-revalidateキャッシュと組み合わせることで、専用のルーターローダーやSWRと同じユースケースをカバーしますが、ボイラープレートは少なくなります。
- **Devtoolsはまだありません**: ルートDevtoolsは計画中です。現在のトレードオフは、型付きルートシステムがコンパイル時の安全性を提供し、ほとんどのルーティングエラーをブラウザに到達する前にキャッチすることです。
- **アウトレットとパラレルルート**: `@lazarv/react-server`は名前付きアウトレット（`@sidebar`、`@content`）を使用し、レイアウトに型付きプロップとしてレンダリングします。これはNext.jsのパラレルルートと機能的に類似していますが、より強い型付けを備えています — 各アウトレットはブランド付きReact要素であり、誤ってアウトレットを入れ替えることを防ぎます。
- **Waku**: WakuはVite上に構築されたもう一つのRSCネイティブフレームワークですが、意図的にミニマルなアプローチを取っています。基本的なファイルベースのルーティングとレイアウト、RSCサポートを提供しますが、型付きルーティングシステム、検索パラメータ処理、スクロール復元、ミドルウェア、その他の高度なルーティング機能は備えていません。Wakuの`Link`コンポーネントは基本的な`scroll`と`prefetchOnEnter`/`prefetchOnView`プロップをサポートしていますが、ルーターAPIのほとんどはまだ`unstable_`マークが付いています。Wakuは高度なルーティングを必要としないシンプルなRSCアプリケーションに適しています。
